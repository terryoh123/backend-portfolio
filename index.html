고객사별 분리 DB 기반 통계 집계 로직 개발

MySQL / 동적 UNION ALL / 클레임 통계 집계
문제 상황
고객사별로 분리된 DB 구조에서 연도별 클레임 건수와 사유를 통합 조회해야 했습니다. 단일 테이블 조회가 아니라, 다수의 고객사 DB에서 조건에 맞는 데이터를 모아 한 번에 집계해야 하는 구조였습니다.
내가 구현한 방식
  • 대상 고객사 목록을 먼저 조회한 뒤 고객사별 DB명을 동적으로 구성
  • 고객사별 `clame_info` 조회 쿼리를 생성하고 `UNION ALL`로 결합
  • 연도별·클레임 사유별로 그룹핑해 전체 합계를 계산
  • PHP에서 연도별 총계 배열을 별도로 계산해 화면 테이블에 함께 노출
핵심 포인트
  • 멀티 테넌트에 가까운 구조에서 통합 통계를 제공하는 집계 로직 구현
  • 클레임 사유가 없는 값까지 예외 처리해 화면 품질을 높임
  • 단순 조회가 아니라 연도별 총계까지 함께 계산하는 형태로 운영 활용도 강화
결과
  • 분산된 DB 구조에서도 운영/분석에 필요한 통합 통계 제공 가능
  • 클레임 이슈를 연도·사유 단위로 빠르게 확인 가능
  • 고객사 상태 분석과 운영 대응에 필요한 근거 데이터 확보
통계/운영 화면과 함께 활용되는 내부 모니터링 페이지 예시
고객사 정보 검색 및 관리 화면: 운영 담당자가 고객 정보를 빠르게 조회·관리할 수 있도록 구성
