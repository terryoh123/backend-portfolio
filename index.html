고객사별 분리 DB 기반 통계 집계 로직 개발

MySQL / 동적 UNION ALL / 클레임 통계 집계
문제 상황
고객사별로 DB가 분리된 구조에서 연도별 클레임 건수와 사유를 통합 조회해야 했습니다. 단일 테이블 조회가 아니라 다수의 고객사 DB를 묶어 운영에 활용 가능한 형태로 집계해야 하는 요구사항이 있었습니다.
해결 방식
  • 대상 고객사 목록을 먼저 조회한 뒤 고객사별 DB명을 동적으로 생성
  • 각 DB의 클레임 데이터를 조회하는 쿼리를 만들고 UNION ALL로 결합
  • 연도별, 클레임 사유별 집계를 수행하고 PHP에서 총계 계산 로직 추가
  • 사유 없음(NULL/빈값) 케이스까지 예외 처리해 운영 화면 품질 개선
기술 포인트
  • 멀티 테넌트에 가까운 분리 DB 구조에서 통합 통계를 제공하는 집계 로직 구현
  • 운영 화면에 바로 활용할 수 있도록 단순 조회가 아니라 요약·총계 형태로 후처리
  • PHP와 MySQL을 조합해 동적 쿼리 생성과 화면용 데이터 구조화를 동시에 처리
성과
  • 분산된 DB 구조에서도 연도별 / 사유별 통합 통계를 제공 가능
  • 고객사 이슈와 클레임 현황을 빠르게 파악할 수 있는 데이터 기반 확보
  • 운영·분석용 내부 페이지의 활용도 향상
고객사 목록 조회 후 대상 DB를 수집하고 통계 집계를 위한 UNION ALL 쿼리를 구성한 코드
연도별 총계 계산과 예외값 처리 로직을 포함한 PHP 후처리 코드
MySQL UNION ALL Data Aggregation PHP Query Design

프로젝트 개요
